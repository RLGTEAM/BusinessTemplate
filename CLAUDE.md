## Claude-specific notes

- After editing `business.json` or `business.schema.ts`, run `npm run validate:content` before anything else — it gives the fastest, clearest error messages.
- `npm run test:e2e` builds and serves the site itself; don't start `npm run dev` alongside it (port 4321 conflict).
- Windows note: write files as UTF-8 **without BOM** — a BOM in `business.json` breaks `JSON.parse` at build time.
- When asked to "re-theme" or "rebrand", touch only `business.json` (`voice.palette` + copy). If that isn't enough, extend `global.css` tokens — never inline colors in components.
- Placeholder images are generated by `npx tsx scripts/generate-placeholders.ts`; real client photos replace files in `src/assets/images/` keeping the same filenames (or update `business.json`).
